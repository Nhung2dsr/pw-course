BÀI 2 - CÁC VÙNG TRONG GIT

Git có ba vùng chính:

1. Working directory

- Working directory là nơi chứa các file mà người dùng đang tạo, sửa hoặc xóa.

- File mới được tạo nhưng chưa chạy git add sẽ nằm trong working directory và có trạng thái untracked.

- File đã được Git theo dõi nhưng vừa chỉnh sửa sẽ có trạng thái modified.

2. Staging area

- Staging area là nơi chứa các thay đổi đã được lựa chọn để chuẩn bị commit.

- Dùng lệnh git add <file-name> để đưa một file vào staging area

- Dùng lệnh git add . để đưa tất cả file vào staging area

3. Repository

- Repository là nơi lưu các thay đổi đã được commit.

- Dùng lệnh git commit -m "message" để đưa các file từ staging area vào repository.

4. Local

- Local là thư mục chưa được khởi tạo Git.

- Khi chưa chạy git init, tất cả file chỉ nằm trong vùng local.

- Sau khi chạy git init, thư mục trở thành một Git repository và các file chuyển sang working directory.
